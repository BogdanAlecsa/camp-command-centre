# Templates

Templates are reusable planning assets.

They are important, but they should not be overbuilt too early.

---

## Template Principle

Templates should help leaders start faster without forcing every camp to be the same.

A template should be copied into a camp, not live-linked.

This means:

- changing a template later does not unexpectedly change an existing camp
- each camp remains self-contained
- leaders can adapt the copied content

---

## Current Template-Like Areas

The app already has some template-like ideas:

- default sections
- task phases/categories
- activity records
- risk assessment structures
- programme session types
- session staff roles
- backup plan reasons

These are not yet a full template library.

## Programme Templates

Programme templates could eventually include:

- weekend camp
- indoor sleepover
- day activity
- group rotation
- expedition
- family camp
- wet-weather activity pack

A programme template should create editable sessions inside the camp.

---

## Task Templates

Task templates could include common camp preparation tasks.

Examples:

- confirm venue
- check permit
- submit NAN
- prepare risk assessments
- prepare first aid kit
- create parent information
- check dietary requirements
- arrange transport
- print registers

Task templates should create real camp tasks once copied.

---

## Risk Assessment Templates

Risk assessment templates should support:

- reusable generic hazards
- reusable controls
- activity-specific controls
- site/event RA coverage
- status and review workflow

Future RA templates need to avoid encouraging blind copy-paste.

The app should help leaders adapt and review risk information.

---

## Backup Activity Templates

Backup plans can already link to existing Activities.

This allows activity records to act as a lightweight backup activity library.

Examples:

- indoor quiz
- board games
- craft challenge
- campfire songs
- shelter-building talk
- knot relay
- silent disco / film night
- wide-game alternative

---

## Deferred Template Library

A full template library is deferred.

Before building it, the app needs stable core workflows for:

- people
- attendance
- teams
- tasks
- activities
- programme
- risk assessments
- print outputs
